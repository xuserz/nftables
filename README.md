# nftables Config with VK Blocking

Набор конфигурационных файлов для nftables с автоматической блокировкой всех сервисов VK (ВКонтакте, OK.ru, max.ru) и защитой SSH.

## 📋 Возможности

- 🚫 **Полная блокировка VK** — все подсети AS47764 (более 130,000 IP)
- 🚫 **Блокировка max.ru** — все IP-адреса сервиса
- 🔒 **Защита SSH** — порт 22 всегда открыт и не может быть случайно заблокирован
- 📦 **Модульная структура** — конфиги разбиты по файлам для удобства
- ⚡ **Простая установка** — один скрипт сделает всё за тебя
- 🎯 **Гибкая настройка** — скрипт спросит какие порты разрешить

## 📁 Структура проекта

```
nftables/
├── main.conf              # Главный конфиг
├── install.sh             # Скрипт автоматической установки
├── blocklists/
│   ├── ipv4.nft          # IPv4 подсети для блокировки
│   └── ipv6.nft          # IPv6 подсети для блокировки
└── rules/
    ├── input.nft         # Правила для входящего трафика
    └── output.nft        # Правила для исходящего трафика
```

После установки на сервере появится:

```
/etc/nftables/
├── main.conf
├── variables.conf         # Создаётся динамически с твоими портами
├── blocklists/
│   ├── ipv4.nft
│   └── ipv6.nft
└── rules/
    ├── input.nft
    └── output.nft
```

## 📝 Процесс установки

Скрипт задаст тебе всего 2 вопроса:

1. **Какие TCP порты разрешить?** (через пробел)
   ```
   Пример: 80 443 42603
   ```

2. **Какие UDP порты разрешить?** (через пробел)
   ```
   Пример: 53 42603
   ```

> ⚠️ **Важно:** SSH (порт 22) уже добавлен автоматически и **не может быть заблокирован** даже если ты не укажешь его.

## 🔧 Что делает скрипт

1. Устанавливает nftables
2. Создаёт структуру каталогов `/etc/nftables/`
3. Скачивает конфиги из GitHub
4. Создаёт `variables.conf` с указанными тобой портами
5. Проверяет синтаксис конфигурации
6. Применяет правила
7. Включает автозагрузку
8. Предлагает перезагрузить сервер

## 📊 Результат после установки

| Тип трафика                | Действие                     |
| -------------------------- | ---------------------------- |
| Входящий с Сервисов        | ❌ Заблокирован               |
| Исходящий к Сервисов       | ❌ Заблокирован               |
| SSH (порт 22)              | ✅ Всегда открыт              |
| Указанные тобой TCP порты  | ✅ Открыты                    |
| Указанные тобой UDP порты  | ✅ Открыты                    |
| Остальной входящий трафик  | ❌ Заблокирован (policy drop) |
| Остальной исходящий трафик | ✅ Разрешён                   |

## 🔧 Ручная настройка

### Добавить новый TCP порт

Открой `/etc/nftables/variables.conf` и добавь порт:

```nft
define ALLOWED_TCP_PORTS = {
    80,
    443,
    8080,      # новый порт
}
```

Затем примени изменения:

```bash
sudo nft -f /etc/nftables/main.conf
```

### Добавить новый IP для блокировки

Открой `/etc/nftables/blocklists/ipv4.nft` и добавь IP/подсеть:

```nft
define block_ipv4 = {
    # ... существующие IP ...
    1.2.3.4,           # новый IP
    192.168.0.0/16,    # новая подсеть
}
```

### Временно отключить блокировку VK

```bash
# Закомментировать строки с block_ipv4 в rules/input.nft и rules/output.nft
sudo nano /etc/nftables/rules/input.nft
sudo nano /etc/nftables/rules/output.nft

# Применить изменения
sudo nft -f /etc/nftables/main.conf
```

## ❓ Частые вопросы

### Что если я заблокирую себе доступ?
SSH (порт 22) защищён отдельным правилом и **не может быть заблокирован** через переменные. Ты всегда сможешь подключиться.

### Как проверить что блокировка работает?
```bash
# Попробовать пропинговать VK
ping vk.com
# Должен быть таймаут

# Проверить через curl
curl -I https://vk.com
# Должна быть ошибка соединения
```

### Блокируются ли поддомены?
Да, блокируются все IP-адреса из подсетей VK, включая все поддомены.

### Нужно ли перезагружать сервер после установки?
Не обязательно, но скрипт предложит это сделать для гарантии.
